Wat is Linked Data

Algemeen

In de huidige tijd is veel data beschikbaar. Veel van deze data kent een geografisch component en is voor potentieel iedereen interessant Bijvoorbeeld voor het implementeren in een webapplicatie. Echter de meeste geo-informatie voldoet aan een standaard die delen niet eenvoudig maakt. Er is dus noodzaak voor een model/infrastructuur dat het makkelijker maakt om deze data op grote schaal te kunnen delen/hergebruiken. Linked Data kan uitkomst bieden om als zo een infrastructuur te dienen. Meer over de technologie Linked Data wordt in de volgende secties nader toegelicht.

Datamanagement geschiedenis

Voor een aantal decennia hebben traditioniële databases een grote rol gespeeld in het beheren,bewaren en het bevragen van data 1. Men bewaart informatie bij relationiele databases in tabellen, waarbij men meestal informatie over verschillende tabellen verspreidt. Om een bepaalde bevraging te doen, waarbij de informatie over verschillende tabellen verspreid zijn, wordt er gebruik gemaakt van een zogenaamde join. Een join operatie wordt over het algemeen als een dure operatie beschouwd. Echter in de huidige tijd, waarbij de eigenschappen van de hardware 2 verbeterd zijn t.o.v. een aantal decennia terug en verder meer noodzaak is voor schaalbaarheid en performance door de opkomst van de zogenaamde big-data systemen, is het eventueel van belang om een andere database-systeem in acht te nemen. Men kan daarbij aan de NoSQL databases denken wat verder verduidelijkt wordt in de volgende sectie.

NOSQL database

De NoSQL databases zijn opgekomen die verschillen in de manier waarop informatie wordt bewaard en bevraagd t.o.v de relationiele databases 1. Binnen de NoSQL databases zijn de volgende vier databases veel voorkomend 1:

  • graph database
  • key-value database
  • document database
  • column family database

Linked Data valt onder een 'graph database' en zal in de volgende secties nader toegelicht worden. De andere 3 databases zullen niet ter sprake komen. Bij een graph database ligt de nadruk om het modelleren van netwerken 3. Zo een netwerk bestaan uit 'vertices' en 'edges'. Vertices betreffen de objecten in het netwerk, waarbij edges de relatie aangeven tussen de objecten. Zie de afbeelding hieronder voor een simpele weergave van een verbinding tussen twee nodes door middel van een edge.

Binnen de graph databases kan er weer onderscheid gemaakt tussen verschillende modellen 4.

  • hypergraphs
  • property graph
  • Resource Description Framework (RDF) graph

Linked Data maakt gebruik van het RDF model en nog andere technologiën.

Het Semantische Web

De relatie tussen het Semantische Web en Linked Data is niet eenduidig. Sommigen beschouwen Linked Data als een losstaand technologie, waarbij anderen Linked beschouwen als een extensie van het Semantische Web en weer anderen beschouwen Linked Data als het Semantische web, maar dan goed uitgevoerd. Hoewel voor het gemak beschouw ik Linked Data als een extensie van het Semantische web, aangezien Linked veel van de achterliggende technologie van het Semantische Web gebruikt. Om te beginnen, zal ik eerst het doel van het Semantische Web toelichten. Het Semantische Web heeft als doel om de betekenis van data machine-vriendelijk 5 te maken. Dit zou ervoor moeten zorgen dat zoekopdrachten beter de data zou moeten kunnen representeren. Sinds een computer niet zelf de betekenis van data kan achterhalen, is het daarom noodzakelijk om een gestructueerde formaat aan de data mee te geven, wat informatie betreft over wat de data inhoudt 5. Het probleem kan bijvoorbeeld wezen, dat een woord op meerdere manieren kan worden opgevat. Daarom is het meegeven van zo'n gestructureerde formaat noodzakelijk.

Hoe kan Linked Data worden beschreven?

Linked Data kan worden beschouwd als een technologie wat als doel heeft om data in een bepaalde formaat te structureren (RDF), zodat deze data met elkaar gelinkt kunnen worden. Dit zou ervoor moeten zorgen dat Semantische bevragingen beter gesteld kunnen worden 6.

Aangezien, men geïnteresseerd is in toepassingen van Linked Data op het gebied van geo-informatie, wordt in de volgende sectie nadrukkelijk iets verteld over de verschillen tussen het Semantische Web en geo-informatie.
