Informatie over de webmap applicatie

In de link webmap applicatie krijg je een een webmap applicatie te zien waarin je de mogelijkheid hebt om drie sparql queries uit te voeren.

  1. Het eerste wat je moet doen, is op een gebouw klikken, waarbij vervolgens alle eigenschappen van het (BRT) gebouw worden opgevraagd met een SPARQL query. Het klikken van een gebouw kan momenteel alleen in de regio Amsterdam aangezien er een webscene is gebruikt voor de webmap applicatie, waarbij alleen gebouwen in de regio Amsterdam in 3d is weergeven. Dit is om de reden dat het maken van 3d BRT gebouwen teveel tijd in beslag zou nemen en daarom als prototype voor de regio Amsterdam was gekozen.
  2. Nadat je op een gebouw heb geklikt, heb je twee andere mogelijkheden om een SPARQL query uit te voeren. Bij de eerst sparql query, kun je CBS gegevens opvragen op wijk of buurt niveau. De output van deze query wordt vervolgens in een diagram gevisualiseerd.
  3. Als laatse is er een query, waarbij je bepaalde type gebouwen kunt opvragen rondom het geklikte gebouw. De gebruiker kan zelf bepalen hoeveel resultaten worden teruggegeven, wat voor type gebouwen het betreft en de afstand tot waar de gebouwen worden meegerekend.

Informatie over het maken van de webmap applicatie

Voor het maken van de webmap applicatie heb ik de API van ArcGIS gebruikt. Meer informatie over deze API is te vinden op de volgende link https://developers.arcgis.com/javascript/. Code van de webmap applicatie is te vinden op Github.

Ik zal proberen ergens binnenkort een tutorial online te zetten om te laten hoe ik de webmap applicatie heb gebouwd.

Waarschuwing: Men is vrij om de code te (her)gebruiken, maar bij maken van de webmap applicatie is geen rekening gehouden met aspecten, zoals beveiliging. Verder zou iemand de webmap applicatie gebruiksvriendelijker of meer esthetisch kunnen maken. Aangezien ik pas een beginner ben op het gebied van JavaScript, HTML en CSS, kan het zich voordoen dat de code die ik heb gebruikt/geschreven niet geheel volgens standaard is. Het is dus geadviseerd om de code waarschijnlijk nog aan te passen/ laten nakijken door iemand met meer ervaring op het gebied van webdevelopment and SPARQL voor daadwerkelijk gebruik.
