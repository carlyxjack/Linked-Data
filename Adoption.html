De adoptie van Linked Data

Toepassingen van Linked Data

Ik baseer dit voornamelijk op mijn interviews die ik heb gehad met Linked Data experts tijdens mijn stage. Ik baseer dit niet dus op literatuur en daarom moeten mensen deze uitspraken eventueel voorzichtig aannemen en eventueel zelf op onderzoek uitgaan. In ieder geval op basis van interviews met een paar Linked Data experts, kwam ik voornamelijk op de volgende toepassingen uit van Linked Data:

  • Het vinden van informatie
  • Het ontdekken van nieuwe informatie
  • Het verhelpen van semantisch conflict

Uitdagingen van Linked Data

Voor de mensen die eventueel geïnterseerd zijn in de adoptie van Linked Data, men zou eventueel bepaalde uitdagingen in acht moeten nemen. Terwijl ik zelf niet veel heb verdiept in de uitdagingen van Linked Data , stuitte ik op een paar uitdagingen die ik heb gevonden in literatuur. Dit zijn slechts de bevindingen van het lezen van een paar artikelen en hoeft dus niet de gehele status quo van de uitdagingen van de adoptie van Linked Data te beschrijven. Op basis van een bepaar bevindingen stuitte ik op de volgende uitdagingen 1:

  • Technische uitdagingen
  • Uitdagingen betreft mentaliteit

De uitdagingen zal ik kort toelichten

Technische uitdagingen

Voor de transformatie van traditional data formaten naar Linked Data, heeft men waarschijnlijk een technische expert nodig 1, aagezien:

  • Veel programma's voor de transformatie van relationiele data naar Linked Data hebben geen Graphical User Interface (GUI) maar maken gebruik van een command line of webservices 1
  • Verder is er voor de transformatie van relationiele data naar Linked data, een transformatie mapping nodig. Deze moet gedefiniëerd worden en verder moet deze transformatie mapping waarschijnlijk aangepast worden per situatie. Wanneer men van verschillende data formaten naar Linked Data zou willen transformeren, dan moet men daar waarschijnlijk verschillende tools voor gebruiken 1.

Een ander belangrijk punt is eventueel de omvang van de data dat getransformeerd moet worden. Maar dit probleem is niet alleen geldend voor Linked Data en is in het algemeen voor alle databases 2.

Uitdagingen betreft mentaliteit en regels

Een artikel beschreef dat in de publieke sector de adoptie van Linked Data langzamer verloopt door de volgende redenen 1:

  • Door bureacratie en regels voornamelijk aanwezig in de publieke sector
  • Geen trainingen aanwezig om over Linked Data te onderwijzen

Eventuele tools voor de transformatie van relational data naar Linked Data

Een artikel beschreef een workbench die bestaat uit verschillende programma's die als doel heeft om de transformatie van relational data naar Linked Data makkelijker te maken. Voor mensen die geïnteresserd zijn in het transfomeren van relationiel data naar Linked Data, kan men eventueel een kijkje nemen op de volgende link: http://linda-project.eu/tools/.

Voor de mensen die eventueel geen relational data naar Linked data zou willen transformeren, maar in plaats daarvan on the fly Linked Data zou willen genereren. Daar is dan de volgende programma voor: ontop. Hierbij werkt het programma dus over de relational database heen en bij het stellen van een SPARQL query, wordt er 'on the fly' triples gemaakt. Meer informatie over de programma ontop kan worden gevonden op de volgende link https://ontop.inf.unibz.it/

Platform Linked Data Nederland heeft verder nog een roadmap beschikbar gesteld wat een overzicht geeft van de stappen die gezet moeten worden om relational data te transfomeren naar Linked Data. Deze roadmap is te vinden op de volgende link: http://www.pilod.nl/wiki/BoekTNO/stappenplan
